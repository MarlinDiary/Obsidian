# 747

我们的题目是「自动生成代码 **结构** 的多样性」，这里的结构多样性指的是解决编程问题的各种变体方法。

为什么要研究问题解法是否有多样性呢？

在传统的计算机教育中，往往会把 **代码的同行评审** 作为提升学生代码能力的一环，在这个过程中我们可以看到别的程序员对同一个问题给出的不同解决方案。如果学生们花时间去理解别人的代码，那么他们会学到更多关于编程的知识以及语言结构的不同用法。从 Bloom’s Taxonomy 的角度看，在这个过程中，学生学会了更高层次的技能，即能*评估* 不同解决方案的质量，从而开始理解如何在有多种解决方案时做出决策。

生成式人工智能其本质上是一个概率语言模型，在生成对同一问题的解答时具有一定程度的随机性，能生成多种解决方案。这就给用大模型生成多种 Example Code 作为教学资源，作为同行评审的重要补充，提升学生的评估能力提供了潜在的可行性。

但我们尚未得知在计算机教育当中，LLM 的变体解决方案在多大程度上能替代和补充传统的同行评审，所以我们的研究就是来量化这种多样程度。

![[001.png]]

至于如何量化呢？以及量化的是哪方面的多样性呢？可以参考以下表格：

| **量化方法** | **量化的是哪方面的多样性** |
| -------- | --------------- |
| 行数 / 长度  | 简洁性             |
| 变量命名     | 命名多样性           |
| AST 树    | 结构差异            |
| 控制流图     | 逻辑路径差异          |

量化后，我们就可以回答：LLM 在多大程度提供了不同的解法？什么样的提示词和模型温度能在保持代码高质量的同时，让学生有机会接触各种解法的代码？
